# Implementation Plan

## Overview

Native iOS (SwiftUI, iOS 26 min) app for a single personal trainer: clients, exercise
library, workout templates, scheduled/recurring sessions, set recording, progression
charts, reminders, calendar push, versioned JSON export/import, biometric lock — all
local, SwiftData behind protocol-typed `@ModelActor` repositories exchanging Sendable
DTOs.

## Conventions & commands

- **Build/test command** (used in every phase; set once):
  ```bash
  export DEVELOPER_DIR=/Applications/Xcode.app/Contents/Developer
  SIM='platform=iOS Simulator,name=iPhone 17'
  xcodegen generate            # regenerate .xcodeproj from project.yml
  xcodebuild -scheme TrainMe -destination "$SIM" build
  xcodebuild -scheme TrainMe -destination "$SIM" test
  ```
  If `xcodegen` is missing: `brew install xcodegen`. If a `sudo` prompt blocks
  `xcode-select`, the `DEVELOPER_DIR` export above is sufficient — no sudo needed.
- **Per-entity file convention**: `Sources/Models/<E>.swift` (`@Model`),
  `Sources/Models/DTO/<E>DTO.swift` (Sendable Codable + mapping),
  `Sources/Persistence/<E>Repository.swift` (protocol + `@ModelActor` impl),
  `Sources/Features/<Feature>/<Name>View.swift` + `<Name>ViewModel.swift`.
- **Testing**: Swift Testing (`import Testing`), in-memory container
  (`makeContainer(inMemory: true)`); heavy repo calls invoked from `Task.detached`.
- **Rules baked in**: no `@Query` in views; only Sendable DTOs cross the actor
  boundary; `append(contentsOf:)` never looped `.append()`; lock on `.background` only.

```swift
@Model final class Client {
    @Attribute(.unique) var id: UUID
    var name: String
    var createdAt: Date
    var updatedAt: Date
    init(id: UUID = UUID(), name: String, createdAt: Date = .now, updatedAt: Date = .now) { … }
}
struct ClientDTO: Codable, Sendable, Identifiable, Equatable {
    let id: UUID; var name: String; var createdAt: Date; var updatedAt: Date
}
extension ClientDTO { init(_ m: Client) { … }; func apply(to m: Client) { m.name = name; m.updatedAt = .now } }

protocol ClientRepository: Sendable {
    func all() async throws -> [ClientDTO]
    func upsert(_ dto: ClientDTO) async throws
    func delete(id: UUID) async throws
}
@ModelActor actor SwiftDataClientRepository: ClientRepository {
    func all() async throws -> [ClientDTO] {
        try modelContext.fetch(FetchDescriptor<Client>(sortBy: [.init(\.name)])).map(ClientDTO.init)
    }
    func upsert(_ dto: ClientDTO) async throws {
        let id = dto.id
        if let existing = try modelContext.fetch(FetchDescriptor<Client>(predicate: #Predicate { $0.id == id })).first {
            dto.apply(to: existing)
        } else { modelContext.insert(Client(id: dto.id, name: dto.name)) }
        try modelContext.save()
    }
    func delete(id: UUID) async throws { … fetch by id, modelContext.delete, save … }
}
```

## Versioning policy (both surfaces)

**Export archive** — schemaVersion starts at `1`.
- *Additive-first*: new optional fields do not bump the version.
- *Breaking change*: freeze the outgoing version's types under `Archive/V<n>/`
  (copy, never mutate), add `Archive/V<n+1>/`, write pure
  `migrate(V<n>) -> V<n+1>`, repoint `CurrentArchive`, and add a golden fixture
  `export-v<n+1>.json`. Old fixtures and their import tests are never deleted.
- Import always: probe version → reject newer → migrate older through the chain.

**SwiftData store** — belt and suspenders because migrations are documented-buggy
(`research.md` Q2):
- `VersionedSchema` defined from first release; lightweight stages preferred.
- `StoreBackup` (P0) copies store files aside on app-version change *before* the
  container opens; a failed migration restores the copies and offers
  import-from-export (P8) as recovery. A failed store migration is recoverable,
  never data loss.

## Cross-phase container registration

Each phase adds its new `@Model` types to `PersistenceController.schemaTypes`. After
adding a model, run `xcodegen generate` only if files/resources changed; model-only
additions need no regen.
